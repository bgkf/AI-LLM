Imports
Bring in LangChain components. One import per capability — LLM, agent builder, tool decorator, prompt template.
Tool Definition
A Python function the LLM can call. The docstring tells the model what it does and when to use it. Can have multiple tools.
LLM
The brain. Reads the prompt, decides whether to call a tool or answer directly. Points at your local llama-server via base_url.
Prompt Template
Four slots:
system — agent's personality & rules
{history} — past conversation (memory)
{input} — the user's question
{agent_scratchpad} — mid-loop reasoning
Agent
Wires LLM + tools + prompt into a single runnable. Not yet executable on its own — just the configuration.
Agent Executor
The runtime loop. Calls the agent, runs any tools it requests, feeds results back, repeats until done. max_iterations prevents runaway loops.
Invocation
Starts the loop with a question. Returns a dict — the final answer is always in result["output"].
Dependency Chain — why this order is required
Tool

Prompt
←

Agent

Invocation
←

No dependencies — order is flexible
Requires — must come after its dependencies
Tool, LLM, and Prompt are
independent of each other.
